# item23. 한꺼번에 객체 생성하기
## 요약
### 속성을 제각각 추가하지 말고 한꺼번에 객체로 만들어야 한다. 안전한 타입으로 속성을 추가하려면 객체 전개({...a, ...b})를 사용
### 객체에 조건부로 속성을 추가하는 방법을 익히기.
객체를 한번에 정의하지 못하고, 제각각 나눠서 만들어야 한다면, 타입 단언문(as)를 사용해 타입 체커 통과
```
const pt = {} as Point;
pt.x = 3;
pt.y = 4;

better
const pt: Point = {
 x: 3;
 y: 4;
}
```
객체 전개 연산자 ...를 사용하여 큰 객체를 한번에 만들기\


# item24. 일관성 있는 별칭 사용하기
## 요약
### 별칭은 ts가 타입을 좁히는 것을 방해한다. 변수에 별칭을 사용할 때는 일관되게 사용해야 한다.
### 비구조화 문법을 사용해서 일관된 이름을 사용하는 것이 좋다.
### 함수 호출이 객체 속성의 타입 정제를 무효화 할 수 있다는 점을 주의하며, 속성보다 지역 변수 사용을 해야 타입 정제를 믿을 수 있다.
별칭의 값을 변경하면, 원래 속성값에서도 변경
객체 비구조화를 이용하여 일관된 이름을 사용할 수 있다.
객체 비구조화의 주의 점.
1. 전체 object의 속성이 아니라 선택적 속성일 경우에 속성 체크가 필요하기 때문에, 타입의 경계에 null 값을 추가하는게 좋다.
2. 차이가 없지만 이름으로 구별하는 것.
